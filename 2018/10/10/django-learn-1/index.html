Django学习笔记（一）

受不了手册和虚拟机分屏放了。。开始写个学习笔记吧。 讲真一开始只是对这个东西感兴趣而已，想着有时间的话学一下，没想到这么快就“被逼无奈”要开始学了。查了一下，发现这玩意儿也是所谓的 MVC 框架，感觉上好像和 PHP 的 CI 框架差不多，那就试试吧。结果，仔细一看，这两种框架差别还是非常大的。CI 里的 MVC ，Models 模型层用来处理数据库，Views 视图层是前端页面，Controllers 控制器是后台代码。但在 Django 中，Views 是后台代码，取而代之的，Templates 模板层则是前端视图，只有 Models 功能是一样的。同样，CI 框架中 URL 只需要按照规则即可访问，Django 中貌似每次都需要自行配置。确实增加了灵活性，但同时也复杂了许多。 下面就开始吧。我使用的开发环境为 ubuntu18.04+python3.6.5+pycharm 。实际上安装非常简单，使用 pip 执行命令 pip install Django 即可。但那天实验室网不太好，几次都连接失败了，只能通过whl来安装了，不过无伤大雅。随后通过 python 命令行可以查看否安装成功，具体如下： 安装成功之后，下一步就是新建项目了。输入命令 django-admin startproject projectname 即可新建项目。进入项目文件夹，输入命令 python manage.py startapp appname 即可新建 app 。在 app 目录中新建 static 文件夹用于存放 js,css 等文件，新建 templates 文件夹用于存放前端页面。最后在 setting.py 的 INSTALLED_APPS 中，添加 app即可完成注册。 浏览项目的目录结构，manage.py 是一个命令行工具，可以实现很多功能。urls.py 是当前 Django 项目的 URL 声明。settings.py 是当前 Django 项目的设置/配置文件。wsgi.py 是一个 WSGI 兼容的 Web 服务器的入口，用来运行 Django 项目。 到这一步，一个项目就差不多新建好了。不过这还早着呢…..感觉这东西不会这么轻易放过我…..怕了怕了……
